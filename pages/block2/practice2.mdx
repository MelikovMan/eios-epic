# Практическое задание. Проект.

## Цель 
Разработать полнофункциональное одностраничное приложение (SPA) на React, демонстрирующее усвоение ключевых концепций современной Frontend-разработки.

Формат выполнения: Индивидуально или в группах по 2-3 человека

## Требования к проекту
### Обязательные технические требования
#### Архитектура и структура проекта
Использование Create React App или Vite

Четкая структура папок (components, hooks, contexts, utils, styles)

Модульная система импортов/экспортов

#### Компоненты и Props
Не менее 10 различных компонентов

Использование функциональных компонентов

Передача данных через props

Деструктуризация props

PropTypes или TypeScript

#### State Management
Использование useState для локального состояния

Использование useContext для глобального состояния

Минимум 2 кастомных хука

Обработка сложного состояния (объекты, массивы)

#### Жизненный цикл и Side Effects
Использование useEffect для работы с API

Обработка загрузки и ошибок

Очистка эффектов (cleanup functions)

Зависимости в useEffect

#### Обработка событий
Контролируемые компоненты форм

Обработка пользовательского ввода

Валидация форм

Дебаунсинг поиска (если применимо)

#### Роутинг
React Router DOM

Минимум 3 маршрута

Защищенные маршруты (если нужна авторизация)

Навигация между страницами

## Варианты проекта 

### Вариант 1: Интернет-магазин
Основной функционал:

Каталог товаров с фильтрацией

Корзина покупок

Поиск товаров

Страница товара

Оформление заказа

Демонстрируемые темы:

Управление сложным состоянием (корзина, товары)

Работа с формами (оформление заказа)

Контекст для глобального состояния

Оптимизация рендеринга

### Вариант 2: Персональный блог
Основной функционал:

Лента постов

Создание/редактирование постов

Комментарии

Категории и теги

Поиск по контенту

Демонстрируемые темы:

CRUD операции

Работа с формами

Компоненты высшего порядка

Кастомные хуки для API

### Вариант 3: Задачник (Task Manager)
Основной функционал:

Создание задач

Фильтрация (все/активные/выполненные)

Поиск задач

Статистика выполнения

Категории и приоритеты

Демонстрируемые темы:

Локальное состояние

Обработка пользовательского ввода

useReducer для сложной логики

Persistence (сохранение в LocalStorage)

### Вариант 4: Медиа-плеер
Основной функционал:

Плейлисты

Управление воспроизведением

Прогресс бар

Поиск медиа

Избранное

### Свой вариант
Требует согласования с преподавателем.
